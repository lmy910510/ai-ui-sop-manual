AI-UI SOP 手册 / PRD 质量治理

需求"洗白"策略

如何把糊涂账变成精准指令 — 让 PRD 对人类友好，对 AI 也能理解

核心理念
🏗️ 建立"编译器"：AI 把 PM 说的话翻译成可执行的数据

PM 喜欢说「差不多」「大概」「你懂的」— 这些词语对人类来说充满了默契，但对 AI 来说等于编译报错。 我们需要在 PM 和 AI 之间建立一个"编译器"，将模糊的人话转换成精准的结构化数据。

1

AI 引导式访谈

PM 说一句模糊需求 → AI 用"访谈问卷"逼出细节

🎯 核心思路

不指望 PM 一次写清楚。AI 主动发问，一步步把答案"榨"出来，最终生成标准结构文档。

📋 流程

👤 PM 输入模糊需求
→
🤖 Requirement Agent
→
❓ 引导式访谈问卷
→
📝 输出结构化 PRD

💡 Requirement Agent 示例 Prompt

System Prompt 
你是一个需求分析 Agent，任务是帮助 PM 把模糊需求转成结构化 PRD。

当 PM 给出需求描述后，你要：
1. 识别出缺失的关键维度（交互流程、边界条件、状态变化、异常处理等）
2. 生成一组针对性访谈问题（每次不超过 5 个）
3. 根据 PM 的回答迭代追问，直到信息完整
4. 最终输出标准格式 PRD，包含：
   - 功能概述
   - 用户故事
   - 交互流程（Mermaid 流程图）
   - 状态定义
   - 边界条件与异常处理
   - 验收标准

注意：
- 问题要具体、可选项化，降低 PM 回答难度
- 如果 PM 说"随便"或"看着办"，给出合理默认值并确认

🔑 访谈问题示例

  • 交互流程：用户点击按钮后，下一步发生什么？是弹窗确认还是直接执行？
  • 边界条件：列表最多显示多少条？超出后是分页还是无限滚动？
  • 状态变化：提交成功后，按钮状态如何变化？需要 loading 吗？
  • 异常处理：网络超时怎么办？显示什么提示？用户能重试吗？
  • 权限控制：这个功能对所有用户可见吗？未登录用户看到什么？
2

截图即协议

PM 截图画红圈 → AI 通过视觉识别推算规格

🎯 核心思路

PM 最擅长的就是"我要这个效果"+ 一张截图。那我们就把截图当作"半成品合同"，AI 负责从图里"挖"出技术规格。

📋 流程

📸 PM 截图 + 红圈标注
→
🔍 Vision AI 解析
→
📐 推算尺寸/间距/颜色
→
✅ 生成规格清单 + 确认

💡 Vision 解析 Prompt

Vision Prompt 
分析这张 UI 截图，提取以下信息：

1. **布局结构**：识别主要区块、层级关系
2. **组件识别**：标注每个 UI 组件类型（按钮、输入框、卡片等）
3. **视觉规格**（如果可推算）：
   - 大致尺寸比例
   - 间距关系（紧凑/宽松）
   - 主色调、强调色
   - 字体层级（标题/正文/辅助文字）
4. **红圈标注区域**：PM 画圈的地方是重点关注区域，详细描述
5. **交互暗示**：从视觉元素推断可能的交互行为

输出格式：
- 结构化 JSON 或 Markdown 表格
- 附上不确定项，标记"需 PM 确认"

📌 最佳实践

  • 让 PM 用红圈标出重点区域，AI 优先解析这些地方
  • 截图最好附带简短文字说明（哪怕一句话）
  • 解析结果生成后，让 PM 过一遍确认，形成"视觉合同"
3

组件库"强行碰瓷"

PM 说的和该用的不一致时，AI 强制对齐组件库

🎯 核心思路

当 PM 描述的 UI 和团队组件库里的标准组件有出入时，AI 主动"碰瓷"—— 把 PM 说的东西往组件库上靠，输出映射清单和差异报告。

📋 流程

💬 PM 描述 UI 需求
→
🗂️ 匹配组件库
→
⚖️ 生成映射 + 差异报告
→
🤝 PM 确认或调整

💡 组件匹配 Prompt

Component Mapping Prompt 
你是 UI 组件顾问，负责将 PM 的需求描述映射到团队组件库。

输入：PM 的功能/界面描述
输出：组件映射清单

规则：
1. 尽量使用组件库中已有的组件
2. 如果没有完全匹配的组件，给出最接近的替代方案
3. 明确标注需要定制开发的部分
4. 输出差异报告，说明 PM 原始描述与推荐方案的区别

输出格式：
| PM 描述 | 推荐组件 | 匹配度 | 备注 |
|---------|----------|--------|------|
| "一个搜索框" | SearchInput | 100% | 直接使用 |
| "带头像的列表" | AvatarList | 80% | 缺少在线状态，需扩展 |
| "炫酷的动画按钮" | Button + Lottie | 60% | 需自定义动画，建议简化 |

🛡️ 价值

  • 减少返工：提前暴露 PM 期望与技术实现的差距
  • 维护一致性：强制使用组件库，避免"野生组件"
  • 知识沉淀：差异报告可以反哺组件库的迭代
